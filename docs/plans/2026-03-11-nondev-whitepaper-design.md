# Non-Developer Whitepaper & Slides Design

## Overview

Rune 백서 및 슬라이드의 비개발자 버전. 기존 개발자 백서와 동일한 논리를 스타트업 전략 기획자 관점의 시나리오로 재구성.

## Constraints

- **타겟**: 비개발자 — 스타트업 전략 기획, 고등학생도 이해 가능한 수준
- **시나리오**: 가격 전략 결정 (침투 가격 vs 프리미엄 포지셔닝)
- **기술 깊이**: FHE 원리 설명 포함, 수식/프로토콜 이름 없이 비유 중심
- **톤**: 읽기 쉬운 에세이. 과도한 확신이나 문체 과시 없이
- **분량**: 자유 (내용에 맞게)
- **언어**: 한글
- **산출물**: 백서 (Markdown), 슬라이드 (reveal.js HTML, self-serving)

## Structure: 3-Act (Problem → Solution → How)

### 1막: 문제 — AI 에이전트와 사라지는 추론

**1.1 도입 시나리오** — 전략 기획자가 AI 에이전트와 가격 전략을 수립하는 30분 세션. 경쟁사 분석, 가격 감수성, 침투 가격 결정까지의 치밀한 추론. 세션 후 남는 것: 슬랙 한 줄 "가격 결정됨 — 월 9,900원".

**1.2 컨텍스트 증발** — 결정은 남고 추론은 사라진다. 규모의 산수 (8명 x 10건/일 = 분기 5,200건). "왜 9,900원?" 질문에 다시 처음부터.

**1.3 기존 접근의 한계** — 회의록/노션 (사후 작성, AI Slop), 개인 에이전트 메모리 (팀 공유 불가), 사내 검색 (문서 검색 ≠ 추론 검색). 비교 매트릭스.

### 2막: 해결 — Rune이 하는 일

**2.1 두 연산** — 캡처(자동 감지 → 구조화 → 암호화 → 저장)와 리콜(관련 주제 → 자동 검색 → 과거 결정 제시). 별도 명령 불필요.

**2.2 시나리오 재방문** — 가격 전략의 캡처 장면 (결정/근거/기각 대안 구조화), 리콜 장면 (2주 후 마케팅 담당자가 가격 근거 질문). Before/After 대비로 체감시키기: 같은 질문에 대해 (A) Rune 없는 에이전트 = 교과서적 일반론, (B) Rune 있는 에이전트 = 팀의 구체적 맥락이 주입된 답변. "첫날 출근한 천재 컨설턴트" vs "모든 회의에 참석했던 팀원"의 차이.

**2.3 에이전트 비종속** — Claude/ChatGPT/Gemini 어디서든 같은 기억. MCP를 USB 포트 비유로. 도구를 바꿔도 기억은 유지.

### 3막: 원리 — 어떻게 가능한가

**3.1 동형암호(FHE)** — 투표함 비유로 도입. 일반 암호화와의 차이 (꺼내서 열기 vs 잠긴 채로 비교). 고등학생 수준.

**3.2 세 구성 요소** — 비서(에이전트+MCP서버), 잠긴 창고(enVector Cloud), 금고지기(Vault). 역할/아는 것/모르는 것 테이블.

**3.3 데이터 이동 경로** — 캡처/리콜을 비유 기반 단계별 서술. 핵심: 평문은 로컬을 떠나지 않음.

**3.4 신뢰 모델** — 4가지 위협을 비개발자 언어로. 창고 털림/배달 가로챔/비서 해킹/금고지기 털림. Vault 침해는 솔직하게 치명적 위험으로.

**3.5 구현 과제** — 시간순 검색 불가, 관련 다단계 결정 자동 결합 불가. 전향적 프레이밍.

### 마무리

요약 + 링크 (GitHub, enVector, CryptoLab)

- **다이어그램 목록**:
  1. 캡처/리콜 플로우 — 비서/창고/금고지기 비유 기반
  2. 세 구성 요소와 데이터 이동 경로
  3. 신뢰 모델 — 누가 무엇을 보는가
